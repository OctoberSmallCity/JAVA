
       
	1.Java是面向对象的;C++可以是面向对象也可以是面向过程的(为了兼容C);
	
	2.Java是具有跨平台性的;(Java代码是直接运行在虚拟机上(Jvm),所以可以跨平台)
	  C++不具有跨平台性,依赖于特定的平台;
	
	3.Java没有指针,引用可以理解为安全指针;
	  C++有指针;
	  
	4.Java运行速度比C++慢;
	 (1.Java编译完需要JVM解释执行;
	  2.运行时类型/引用/溢出的检测;
	  3.字节码加载执行的开销;
	  4.GC垃圾回收的开销;
	  5.Java采用动态链接,用到哪个类再加载到内存中;C++使用静态链接,全部先加载;)
	
	5.Java采用Gc自动垃圾回收方式,C++采用析构函数手动回收方式;
	  (C++11中引入智能指针,采用引用计数法进行垃圾回收);
	  
	6.Java不支持多重继承;(×)采用多接口的方式实现;实现多重接口,不是扩展(继续)什么类，而是实现一个类，在其中添加一些属性和行为;
	  C++支持多重继承;
	  
	7.Java不支持操作符重载;
	  C++支持;
	  
	8.Java内置了线程支持;
	  C++只能依靠第三方类库;
	  
	9.Java保留了goto保留字,但不可使用;
	
	
	10.Java不支持条件编译;
	   C++支持通过#ifdef #ifndef 等预处理命令从而实现条件编译;
